# VLM-SAE Pipeline (Per-Model, 3-Seed)

End-to-end Docker package: train modality-specific TopK SAEs on CC3M
embeddings of one VLM, then evaluate on COCO retrieval, ImageNet zero-shot,
cross-modal steering, MMS, and MS — repeated across 3 seeds and aggregated
into mean ± std tables and figures.

## Models

| Key          | Backend       | HF model id                                  | dim  |
|--------------|---------------|----------------------------------------------|------|
| `clip_l`     | transformers  | `openai/clip-vit-large-patch14`              | 768  |
| `openclip_l` | open_clip     | `ViT-L-14` / `datacomp_xl_s13b_b90k`         | 768  |
| `siglip_l`   | transformers  | `google/siglip2-large-patch16-256` *(gated)* | 1024 |

To extend: edit the registry at the top of
`scripts/run_pipeline_per_model.sh`.

## Quick start

```bash
# 1. Build
docker build -t vlm-sae-pipeline -f Dockerfile.vlm_large .

# 2. Run one model × 3 seeds. Persists cache/outputs/logs to host volumes.
docker run --rm --gpus all \
    -e HF_TOKEN=$HF_TOKEN \
    -e MODEL=siglip_l \
    -e SEEDS=0,1,2 \
    -v $PWD/cache:/workspace/lvlm_hallucination/cache \
    -v $PWD/outputs:/workspace/lvlm_hallucination/outputs \
    -v $PWD/.log:/workspace/lvlm_hallucination/.log \
    vlm-sae-pipeline
```

> `siglip_l` requires accepting the EULA at
> <https://huggingface.co/google/siglip2-large-patch16-256> first.

To run all three sequentially:

```bash
for M in clip_l openclip_l siglip_l; do
  docker run --rm --gpus all \
      -e HF_TOKEN=$HF_TOKEN -e MODEL=$M -e SEEDS=0,1,2 \
      -v $PWD/cache:/workspace/lvlm_hallucination/cache \
      -v $PWD/outputs:/workspace/lvlm_hallucination/outputs \
      -v $PWD/.log:/workspace/lvlm_hallucination/.log \
      vlm-sae-pipeline
done
```

## Pipeline (per model)

1. **Extract caches** (idempotent: skips if `meta.json` already exists).
   * COCO Karpathy (val + test, 5 captions × image)
   * ImageNet val (50k images + 80 templates × 1000 classes)
   * CC3M val (~13k pairs, streaming)
   * CC3M train (capped at `CC3M_MAX_SAMPLES=1_000_000` by default; full = 2.87M)
2. **Preprocess** CC3M train cache → stacked tensors for mmap-loading.
3. **3-seed sweep** (`bash scripts/run_seed_sweep.sh cc3m_<MODEL> <SEED>`):
   * Train 5 SAE variants (Shared, Modality-Specific, Iso-Energy, Group-Sparse, Post-hoc).
   * COCO retrieval + recon, ImageNet zero-shot + recon.
   * Cross-modal steering @ α ∈ {0, 0.1, 0.25, 0.5, 1.0, 2.0}.
   * Multimodal Monosemanticity Score (MMS) + Pach MS (image / text).
4. **Aggregate** N=3 seed outputs into mean ± std `table.{md,tex}`,
   hammer-bar steering plot, and MS / MMS sorted-curve plots.

## Outputs (host paths)

```
cache/<prefix>_{cc3m, cc3m_val, coco, imagenet}/   # reusable embedding caches
outputs/real_exp_cc3m_<MODEL>_s{0,1,2}/             # per-seed runs
outputs/real_exp_cc3m_<MODEL>_mean/
    table.md                                        # mean ± std (N=3)
    table.tex
    cross_modal_steering/{<variant>/summary.json, steering_map_bar.{pdf,png,svg}}
    ms_curves.{pdf,png,svg}
    mms_curves.{pdf,png,svg}
.log/                                               # all stdout/stderr
```

## Hardware / disk

* **GPU**: single device, ≥16 GB VRAM (training peaks ~10 GB at L=8192, batch=1024).
* **Disk**: ~30 GB per model (cache ~12 GB + ckpts/evals ~5 GB + logs).
* **CPU**: ≥2 cores recommended (CC3M streaming bound by JPEG-decode).
* **Time** *(single A100, 1 model, 3 seeds, CC3M cap 1M)*:
  | Step                       | wall  |
  |----------------------------|-------|
  | extract caches             | ~9 h  |
  | preprocess + 3-seed sweep  | ~12 h |
  | aggregate                  | < 5 m |
  | **total**                  | ~21 h |

## Env tunables

| Var                 | Default | Meaning                                 |
|---------------------|---------|-----------------------------------------|
| `HF_TOKEN`          | —       | HF access token (required)              |
| `MODEL`             | siglip_l| Model key from registry                 |
| `SEEDS`             | 0,1,2   | CSV of seeds to run                     |
| `CC3M_MAX_SAMPLES`  | 1000000 | Cap on CC3M train extraction            |
| `CC3M_NUM_WORKERS`  | 2       | DataLoader workers for CC3M streaming   |
| `BATCH`             | 1024    | SAE training batch size                 |
| `K`                 | 32      | TopK                                    |
| `LATENT`            | 8192    | SAE width                               |
| `EPOCHS`            | 10      | CC3M training epochs                    |

(Training hyperparameters live in `configs/real/cc3m.yaml`; the per-model
config is generated by sed-templating that file with the model's
hidden_size and cache paths.)

## Resume / re-run

Every step is idempotent. If extraction or training is interrupted, just
re-run with the same `MODEL` + `SEEDS` — completed caches and final
checkpoints are detected and skipped. To force re-extraction, delete
the corresponding `cache/<prefix>_*/meta.json`.

## Failure modes

| Symptom                         | Fix                                                           |
|---------------------------------|---------------------------------------------------------------|
| `401 Unauthorized` from HF      | `HF_TOKEN` missing or model EULA not accepted                 |
| `OSError: ... gated`            | Accept SigLIP2 EULA on its model page                         |
| CUDA OOM during training        | re-run with `BATCH=512`                                       |
| `No module named 'open_clip'`   | already in image; if running natively: `pip install open_clip_torch` |
| CC3M extraction extremely slow  | bound by CPU — increase host CPU allocation or `CC3M_NUM_WORKERS` |
